# Repository Guidelines

Conventions for any agent (or human) working in this repo.

## Keep companion files in sync with code

When a change touches the CLI surface, behavior, report format, or error messages, update
these in the **same** change — don't leave them for later:

1. **Skill** — the Portico skill is the agent-facing manual. Update the canonical skill body
   at `packages/skills/portico/SKILL.md` whenever commands, flags, report sections, or
   troubleshooting guidance change. That file is the single source; the `.claude/` and
   `.agents/` copies are gitignored and regenerated by `portico init`, so never edit those
   directly.

2. **Docs & README (English only)** — update the English docs under `docs/` and `README.md`
   to match the new behavior or CLI surface. The `*.zh-CN.md` mirrors do **not** need to stay
   in sync; update the English versions only.
